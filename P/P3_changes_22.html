P3 Design changes

Link to our website: Torsdagsbar

Our changes

While working with the code we have had to make some changes of are design. There are different reasons for the changes and the changes are both covering style and structure. Here you can see a list of our changes.

How we fulfilled our minimum requirements

  • "You can always come back to the front page."

    We have made the logo clickable to bring you back to the frontpage at any time. This is a pattern familiar to users, and therefore follow the universal design guidelines.

  • "Include a nav-bar"

    Mathilde made sure of this! It is responsive to the device size.

  • "You can find information about how to pay for drinks at the events"

    This has been included by Ane in the digital drinking-cards page.

  • "You can always come back to the frontpage in one click."

    Again - Mathilde nailed it! The navbar sticks to the top of all pages.

  • "Every main page should be easily accessed through the nav-bar."

    Mathilde again. Our hero.

  • "We simulate that the user buys “drinking cards” to use as payment for drinks at the events."

    Ane has mimicked this easy transaction on the digital drinking cards-page. We didn't use the vipps API as this is beyond the skills we learn in this course. Know your limits.

Who worked on what?

Mathilde worked on the homepage, CSS and navbar. Ane worked on the about us page, as well as the digital drinking cards. Solveig worked on the drinkspage, as well as the hover-animation artwork of the drinking glass. Anna made the join us page with the integrated form, and also worked on making the front page responsive. Julie worked on the upcoming events page, and made the footer. Everyone added individual parts to the CSS after the design system was implemented - firstly as a style in each individual page, and at last we combined it all to the same CSS-page. For the Javascript document, we put the header and footer in the main script, and made individual scripts on each page.

Modularity requirements

The modularity requirement was handled in that every person made sure their page was responsive. Especially the pages with a photograph with a text container to it’s right, had a bigger job to do here in structuring it for smaller devices. The footer and header was made responsive in Javascript, and was of course used on all the pages as well.

Specific changes to the pages:

  • Changes to about.html

    We did not make too many changes from the original plan on this page. We didn't include a picture of the people running Torsdags Bar because we couldn't find one. Instead we put a picture from the muppet show because it matched our graphic profile and made the page a bit more humorous.

    We did not include an extra link to the “join us” site because the navbar already makes it easy to find at any time, so an extra link seemed unnecessary.

  • Changes to digital_drinking_cards.html

    We did not include the vipps API as this is outside the scope of this class, and instead simulated a purchase in an easy way.

    We also moved the placement of the loading drinks glass hover-animation to the top, next to the title.

  • Changes to drinks.html

    We made the drinktitles clickable to the recipe page from where we found them, as the hover-effect color made them look clickable. This was a great opportunity to link up the recipes!

    Made small changes to the font-sizes in order to make the hierarchy of text understandable and information more accessible.

    We also swapped one of the original drinks with another one in the same style as the others, to make the photos carry the same look.

  • Changes to index.html

    We added arrows to navigate the image carousel for a more interactive experience for the user.

  • Changes to join_us.html

    We did not make any drastic changes from the original plan on this page. We chose a joyful picture, wrote a little text about Torsdagsbar and made a form where you get in contact with Torsdagsbar.

    We make some small changes compared to the Figma wireframes; picture and font size.

  • Changes to upcoming_event.html

    We made some small changes to the design because it was easier to see what was missing when we made the real prototype.

    We added a border around each event to make sure it was easy to separate them.

    We also scaled up the pictures.

  • Changes to CSS styling

    The font sizes are changed, in order for the sites to look nice.

    h1 is now 60px instead of 64px and h2 is 20px.

    We also decided to not include the h3 font, since we didn’t feel we had the need for it.

  • Changes to script.js

    404 this page does not exist anymore. Instead we moved the footer into one page called footer.js and header.js connected to each page.

  • Changes to reqiered javaScript 1: photo carousel

    The photo carousel was almost made the way we planned. Instead of scrolling through pictures, you can click your way through them.

  • Changes to reqiered javaScript 2: Loading icon

    Instead of making a loading-icon connected to the purchase page, we made a hover-animation connected to the drinks-page, to give the user some fun interaction and the page some more graphic elements.

  • Changes to reqiered javaScript 3: "Time to next event"

    We changed the concept from having a box that would show the next event depending on what date it is, to having a countdown on each upcoming event on the “upcoming event”-side. We did not want to show this in separate boxes, but instead write “Days left:...” in the bottom of the boxes of each event.

  • Changes to reqiered javaScript 4: Join us form

    Instead of the planned “Animation effect while on headings” we made a form on the join us page. We found that a form would be better for our webpage.

    When the “Apply here” button is pressed, a form poppes up, where you can write your name and email. In reality this will be sent to a person in the Torsdagsbar team.
